# Managing your Storage Allocations on the RDF-Active

Your RDF-Active storage allocation is primarily managed using [RECAP](../../recap/index.md). This page contains points on how to carry out certain tasks using RECAP.

## How do I view...

### Storage allocations I'm a member of?

See the [Viewing your Allocations](../../recap/allocations.md#viewing-your-allocations) page which explains how to view your allocations.

### Details of a storage allocation?

See the [Viewing details of an allocation](../../recap/allocations.md#viewing-details-of-an-allocation) page which explains how to view the details of an allocation.

### The start and end date of an allocation?

At the top of the allocation details page, you'll find summary information of the allocation including the start and end date of the allocation.

### The requested quota for the storage allocation and where it is located on the file system?

The Allocation Attributes section contains details of the allocation such as:

* Storage and Files quota, in TB and number of files.
* The location of the allocation on the file system.
* The agreed shortname of the allocation.

### How much space I have left in my storage allocation?

The Allocation Attributes section contains details of your usage (disk space and number of files) against the storage quota.

## How do I add or remove users from my storage allocation?

A user must be added to your [Research Group](../../recap/research-groups.md) in RECAP before they can be added to the storage allocation; [adding a user to a Research Group](../../recap/research-groups.md#adding-users-to-your-research-group) will provide an option to add the user to any RDF-Active allocations at the same time. 

If a user is already a member of your Research Group, you can following the guidance on [Adding a user to an allocation](../../recap/allocations.md#adding-a-user-to-an-allocation) to provide them access to the storage allocation.
