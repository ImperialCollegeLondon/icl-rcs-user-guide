# Managing your Storage Allocations on the RDF-Active

Your RDF-Active storage allocation is primarily managed using our [ReCAP](../../recap/recap-index.md) service. This page contains a quick overview on how to carry out common tasks using ReCAP. Please give these a skim to gain an understanding of what you are and aren't able to do.

Reviewing the wider [ReCAP allocation](../../recap/allocations.md) documentation may be of use.

/// details | How do I view the storage allocations I have access to?
    type: faq
Viewing the allocations you have access is a straight forward process that [we have documented](../../recap/allocations.md#viewing-your-allocations) elsewhere.
///

/// details | How do I view the details of a storage allocation?
    type: faq
See any of the [multiple methods](../../recap/allocations.md#viewing-details-of-an-allocation) we've documented to view the details of allocations you have access to.
///

/// details | What details of a storage allocation are saved?
    type: faq
Once you have navigated to the details page of an allocation, you should have a summary of the following information:

* Storage and Files quota, in TB and number of files.
* The location of the allocation on the file system.
* The agreed shortname of the allocation.
///

/// details | How do I view the start and end date of an allocation?
    type: faq
At the top of the [allocation details page](../../recap/allocations.md#viewing-details-of-an-allocation), you'll find summary information of the allocation including the start and end date of the allocation.
///

/// details | How do I view the requested quota for the storage allocation and where it is located on the file system?
    type: faq
The Allocation Attributes section contains details of the allocation such as:

* The associated research group.
* A text explanation for why the allocation exists, dubbed the "Justification".
* Start and end dates, as well as the date created, the date last modified, and the expiry/renewal date.
* The file quota, referring to the total amount of individual files allowed -- regardless of file size -- and the total amount of individual files present in the allocation.
* The GID.
* The file path.
* The agreed shortname of the allocation.
* The storage quota, referring to total storage available in terabytes and the amount in active use.
* Users with access to the allocation.
///

/// details | How do I view how much space I have left in my storage allocation?
    type: faq
See the Allocation Attributes section, where you will clearly see the storage and file in the storage allocation against the full quotas.
///

/// details | How do I add or remove users from my storage allocation?
    type: faq
If you are a manager of a research group, then you will be able to [add users to that research group](../../recap/research-groups.md#adding-users-to-your-research-group) and set which of the associated allocations they have access to.

If a user is already a member of a research group, then their access to specific storage allocations can be [added](../../recap/allocations.md#adding-a-user-to-an-allocation) or [removed](../../recap/allocations.md#removing-a-user-from-an-allocation).
///
